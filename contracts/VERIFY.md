# Verifying Qantara contracts on QIE Mainnet

## Reproducible build

```bash
git clone <this-repo>
cd contracts
npm install
npx hardhat compile
```

Compiles with Solidity 0.8.24, optimizer enabled (200 runs), evm target paris, and produces identical bytecode.

Before publishing a release artifact, regenerate and validate the one-file
verification bundle from the checked-in deployment manifests and the current
build artifacts:

```bash
npm run build
node scripts/regen-verified.cjs --check
node scripts/regen-verified.cjs
node scripts/check-verified-manifest.cjs
```

The validator fails if any Qantara contract is absent, an address does not match
the deployment manifests, compiler settings drift, bytecode differs from the
current build artifacts, or source content differs from this repository.

## Compare built bytecode to on-chain

Save as `scripts/verify-onchain.ts` (or run inline with `node -e`):

```ts
const { ethers } = require('ethers');
const fs = require('fs');
require('dotenv').config();

const TARGETS = [
  ['Qantara', '0x27815fC2021345EB38B68D9C8F08679A4aeee030'],
  ['QantaraMultiPay', '0x72a5B88063E5783954c64244b75f9F8fDb3751Bb'],
  ['MilestoneEscrow', '0x1D096D48d7bb2E6eF2FAfD1eC13C867b5461BA98'],
  ['RecurringScheduler', '0xb05D67901A43644fD831EEC4e6554970e791F690'],
  ['BatchPayout', '0x49EC10ACd67716e35E5129c9ae83C62456502a83'],
  ['QantaraChat', '0x76E618ecca8D97038Ec11641E16b9e16a378576A'],
  ['QantaraSplits', '0xBbaeF9CF47C31436505E46cF2a39636a76C7C413'],
  ['QantaraSubscriptionV2', '0x30ACe939BD62b6a9E9aF3f5AB4287b5FB5F39c06'],
  ['QantaraGasRelay', '0xE027abFb3F845c6798fA247f1053Bd1B143768d2'],
];

(async () => {
  const p = new ethers.JsonRpcProvider(process.env.QIE_RPC_URL);
  for (const [name, address] of TARGETS) {
    const artifact = JSON.parse(
      fs.readFileSync(
        fs.existsSync(`artifacts/contracts/${name}.sol/${name}.json`)
          ? `artifacts/contracts/${name}.sol/${name}.json`
          : `artifacts/contracts/v4/${name}.sol/${name}.json`,
        'utf8',
      ),
    );
    const onchain = await p.getCode(address);
    const match = onchain === artifact.deployedBytecode;
    console.log(name, address, match ? 'MATCH' : 'MISMATCH');
  }
})();
```

All Qantara contract rows should print `MATCH`.

## One-file verification bundle

`deployments/qieMainnet.verified.json` contains, for each contract:
- Live address on chain 1990
- Constructor arguments from the deployment manifest, or the manifest deployer
  when reading earlier deployment records
- Full Solidity source
- Source SHA-256 when regenerated by `scripts/regen-verified.cjs`
- Compiler settings
- ABI
- Init bytecode + deployed bytecode

A third party can diff their artifact JSON against this file byte-for-byte without needing this repo at all.

## When official verification becomes available

If explorer.qie.digital adds Sourcify or Etherscan-style verification, add the
explorer endpoint to `hardhat.config.ts` under an `etherscan` or `sourcify`
block, then verify each deployed address with the matching constructor arguments
from `deployments/qieMainnet.json`, `deployments/qieMainnet.v15.json`, and
`deployments/qieMainnet.v4.json`.

```bash
npx hardhat verify --network qieMainnet 0x27815fC2021345EB38B68D9C8F08679A4aeee030
npx hardhat verify --network qieMainnet 0x72a5B88063E5783954c64244b75f9F8fDb3751Bb
```

For constructor-aware explorer UIs, use the
`verification.<ContractName>.constructorArgs` array from the deployment
manifest. If that field is absent in an earlier deployment record, use the
record's `deployer` address as the only constructor argument. Do not publish a
release until `node scripts/check-verified-manifest.cjs` passes after
compilation.
